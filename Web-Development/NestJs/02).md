Indexes make these queries O(log n) instead of full table scans.

When not to add an index
Do not index:
* Free-text fields (name, work_email)
* Columns rarely used in filters
* Columns with extremely low cardinality if table is tiny

Cost of indexes (why not index everything)
* Slower INSERT / UPDATE
* More disk usage
* More maintenance overhead
So indexing is a performance decision, not a schema requirement.

Rule of thumb
Index columns you filter, group, or join on regularly.
Your usage of index() here is correct and practical, not overengineering.
